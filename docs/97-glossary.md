# 97 - Glossary

- **Adapter:** Platform/source-specific component implementing a common interface.
- **Canonical job:** Normalized representation independent of source.
- **Codex:** Development agent used to implement the repository; not the runtime job-scoring API.
- **Confirmation token:** Short-lived, single-use authorization for one exact external write.
- **Deterministic filter:** Code/rules whose result is repeatable without an LLM.
- **Evidence chunk:** Verified portion of portfolio/case-study data eligible for retrieval.
- **Human-in-the-loop:** User must review/approve at a critical boundary.
- **Idempotency:** Repeating an operation produces no duplicate effect.
- **LLM gateway:** Provider-neutral interface for model calls and controls.
- **Manual capture:** User supplies job URL/text/file instead of automated retrieval.
- **Policy registry:** Machine-enforced permission record per platform/action.
- **Proposal revision:** Immutable version of a generated or edited proposal.
- **RAG:** Retrieval-augmented generation using relevant private evidence.
- **Raw job:** Unmodified source payload stored for traceability.
- **Shadow mode:** Agent runs and evaluates without assisting real submissions.
- **Source cursor:** Pagination/checkpoint used to resume ingestion.
- **Structured output:** Model response constrained to a validated schema.
