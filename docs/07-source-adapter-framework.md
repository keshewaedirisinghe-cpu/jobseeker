# Milestone 07 - Source Adapter Framework

## Goal

Build the plug-in framework for APIs, RSS/JSON feeds, email/manual imports, and future permitted connectors without mixing platform code into the pipeline.

## Adapter responsibilities

An adapter may:

- authenticate through approved methods;
- fetch within allowed limits;
- translate pagination/cursors;
- return raw immutable items and source metadata;
- expose health and capability information;
- classify retryable versus permanent errors.

An adapter may not:

- score jobs;
- generate proposals;
- bypass access controls;
- submit applications unless it implements the separate submission interface and policy permits it;
- store secrets in logs or records.

## Required interfaces

- `SourceAdapter`
- `ManualImportAdapter`
- `FeedAdapter`
- `OfficialAPIAdapter`
- `SourceRegistry`
- `RateLimiter`
- `CursorStore`

## Discovery batch contract

A batch contains:

- platform/source ID;
- adapter version;
- requested and completed time;
- policy version;
- cursor before/after;
- raw item count;
- per-item external ID/URL/payload;
- warnings and partial-failure metadata;
- idempotency key.

## Error taxonomy

- `PolicyDeniedError`
- `AuthenticationError`
- `RateLimitedError` with retry-after
- `TransientSourceError`
- `PermanentSourceError`
- `MalformedPayloadError`
- `SourceChangedError`

A source layout change must disable that adapter after a threshold instead of silently producing empty results.

## Manual import formats

Support universal fallbacks early:

1. Paste title, URL, description, budget, and source.
2. Import a Markdown or text job brief.
3. Import CSV with canonical headers.
4. Optional browser “share/copy” payload generated by the user, not background scraping.

## Observability

Each run records duration, item count, new/duplicate/error count, cursor, HTTP status summary, and policy decision. Never log full authentication headers.

## Tests

Create a fake adapter and fixture-based tests for:

- pagination;
- cursor resume;
- rate limiting;
- retries;
- policy denial before network call;
- malformed item isolation;
- idempotent re-run;
- source-change circuit breaker.

## Required deliverables

- framework modules;
- fake and manual adapters;
- source registry;
- CLI commands `sources list`, `sources test`, and `sources ingest`;
- adapter developer guide.

## Codex execution prompt

```text
Implement Milestone 07 only. Create the typed source adapter framework, source registry, cursor handling, rate limiter, error taxonomy, and universal manual import adapter. Add fixture-based tests and prove a policy denial occurs before any mocked network call. Do not add live public-feed adapters yet.
```

## Acceptance criteria

- [ ] A new source can be added without editing pipeline core.
- [ ] Manual import works end to end into raw storage.
- [ ] Re-running the same batch is idempotent.
- [ ] Rate-limit and source-change behavior are tested.
- [ ] Policy enforcement happens before network access.
